# Design Context — cagricimen.dev

## Project

Personal publishing site for **Çagri Cimen** at https://cagricimen.dev. Built with Astro 6 + bun, deployed to GitHub Pages. Custom hybrid (no Starlight) with four content collections (`writing`, `notes`, `guides`, `projects`) sharing a topic taxonomy.

## Mental model

> **Personal site first, knowledge vault second, docs third.**

The site should feel like a writer's workspace and second brain — not like a developer documentation portal. The architectural cohesion was deliberately chosen over Starlight because Starlight makes everything feel like docs.

## Audience

- Other engineers and AI practitioners
- People interested in how to actually work with AI coding tools, models, and team workflows
- Visitors arriving from links, social, or search — not authenticated users

## Use cases

- Cagri's own second brain — go back to find things he wrote down, stay coherent on what he believes about tools/models
- Public sharing — publish essays, notes, guides, and project case studies
- Cross-collection discovery via topic hubs (one taxonomy applied as tags across all four content modes)
- AI-assisted authoring loop — every piece of content is `.md`/`.mdx` with frontmatter; scaffolders create new entries

## Tone & personality

- **Personal but precise.** Rough notes are okay but they should feel intentional, not lazy.
- **Editorial, not corporate.** Voice over volume.
- **Honest about maturity.** Every entry has a `maturity` field (`seed | growing | stable | archived`) so readers know if they're reading a rough note or a stable guide.
- **Quiet confidence.** No marketing-speak, no hype, no "ship faster with AI" energy.

## Visual direction (current commitment)

- **Aesthetic:** TBD — currently undefined, hence the visual blandness flagged in the audit. Three candidate directions surfaced for evaluation:
  1. **Editorial** (Stripe Press / Reader View vibe) — warm paper background, serif display, asymmetric layout, generous margins
  2. **Brutalist-utilitarian** (Bret Victor / Maggie Appleton) — mono-everything, hard color blocks, no rounded corners, strong typographic contrast
  3. **Warm minimal** (Linear marketing / Ben Mizrahi) — off-white tinted background, one warm accent (terracotta/burnt orange), single beautiful display font, soft motion
- **Direction not yet committed.** `/normalize` filled in the missing structure consistent with the existing minimal token system but did NOT introduce new aesthetic choices. Subsequent commands (`/bolder`, `/colorize`, `/typeset`) should commit to one of the directions above (or another) before doing aesthetic work.

## Constraints

- **Stack:** Astro 6 + MDX + bun + Pagefind + GitHub Pages. No React, no Tailwind, no UI library.
- **Type:** TypeScript strict. Zod schemas for content collections.
- **Build:** `bun run build` chains check → astro build → pagefind → emit-aliases. Don't break this.
- **Accessibility:** WCAG 2.1 AA target. Keyboard, focus-visible, reduced-motion, system theme preference all matter.
- **Performance:** Static site, content-first, fast on cheap hardware. No client-side framework.
- **Responsive:** Mobile-first. Currently the header is broken on mobile (no breakpoints) — this is a known gap to fix in a follow-up `/adapt` pass.

## What NOT to do (anti-patterns specifically called out for this project)

- ❌ Don't use Inter font — it's the AI-default, listed in the frontend-design skill anti-patterns
- ❌ Don't use pure black/white. Tint the neutrals.
- ❌ Don't wrap everything in cards. Vary surfaces.
- ❌ Don't nest cards inside cards (the topic chip on a card on a tinted background problem)
- ❌ Don't go back to Starlight or any pre-built theme. The custom architecture is deliberate.
- ❌ Don't introduce glassmorphism, gradient text, or other 2024-vintage AI tells.
- ❌ Don't use ☀ / ☾ Unicode for the theme toggle — it renders as a near-invisible asterisk on macOS Safari. Use real SVG icons.

## Differentiation

What should someone remember about this site?

- The shared topic taxonomy that aggregates across writing/notes/guides/projects
- The maturity badge — Cagri is honest about how rough or stable each piece is
- A reading experience that feels personal, not corporate
- A site that doesn't try to look like every other AI-generated developer blog

## Design system status

- **Tokens defined:** colors (hex, will migrate to OKLCH in `/colorize`), typography (Inter only, will swap in `/typeset`), spacing (one-off, no scale), width (`--width-text`, `--width-content`, `--width-wide` added in `/normalize`)
- **No motion/easing/elevation tokens yet** — to be added in `/animate`
- **No shadow system yet**
- **Component library:** small custom Astro components in `src/components/`. EntryCard dispatcher (added in `/normalize`) is the canonical pattern for cross-collection lists.
